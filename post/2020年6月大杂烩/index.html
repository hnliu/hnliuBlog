onSaveInstanteState

这个方法主要是当系统想要异常的销毁我们的activity的时候会调用，这是安卓系统的责任，需要系统需要提供给我们一个时机让你可以保存你的数据。比如横竖屏切换的时候，系统回收内存的时候。

activity 横竖屏切换的时候，首先会回调到 onSaveInstanceState 这个方法，表示屏幕马上要进行一个横竖屏的切换动作，这个时候可以把一些需要保存到数据保存到 onSaveInstanceState 方法的参数变量 outState 中，然后在横竖屏切换成功以后，首先 activity 会重新创建，这个时候完全就是一个新的 activity对象，会重新走 onCreate，onStart，onResume 流程，然后 onRestoreInstanceState 回调方法会在 onStart 之后，onResume 之前调用。

下图为activity横竖屏切换的时候，activity生命周期的变化情况，以及ViewModel在这种情况下的一个生命周期变化情况。

将view的引用（activity 和 Fragment）传递到 ViewModel 是一个严重的安全隐患。假设 ViewModel 中请求了网络数据，数据在之后的一段时间返回。此时 view 的引用可能被回收或者不可见，这样就可能产生了内存泄漏或者崩溃。

新建远程分支流程

1. git branch -b  branch_feature_1_6_3      新建本地分支
2. git push origin branch_feature_1_6_3:branch_feature_1_6_3      把本地分支 push 到远程
3. git branch --set-upstream-to=origin/branch_feature_1_6_3 branch_feature_1_6_3      关联远程分支和本地分支

分支合并

1. git rebase -i dev 可以将dev分支合并到当前分支
2. git rebase –abort 放弃一次合并

这里的 -i 是指交互模式。就是说你可以干预 rebase 这个事务的过程，包括设置 commit message，暂停 commit 等等。

isAssignableFrom 方法详解

有两个Class类型的类象，一个是调用isAssignableFrom方法的类对象（后称对象a），以及方法中作为参数的这个类对象（称之为对象b），这两个对象如果满足以下条件则返回true，否则返回false：

a对象所对应类信息是b对象所对应的类信息的父类或者是父接口，简单理解即a是b的父类或接口

a对象所对应类信息与b对象所对应的类信息相同，简单理解即a和b为同一个类或同一个接口
